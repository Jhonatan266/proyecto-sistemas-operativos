Fase 1: Virtualización con Linux

Ver Entregable

Objetivo de la Fase 1

Instalar y configure dos máquinas virtuales Linux (Pop!_OS y Rocky Linux), realizando particionamiento manual del disco, configuración de red y acceso remoto mediante SSH, con el fin de comprender y aplicar conceptos fundamentales de virtualización, administración de sistemas Linux y comunicación entre equipos virtuales.

Para empezar, montamos y aprovisionamos las dos máquinas virtuales de manera independiente usando el hipervisor Oracle VM VirtualBox. A cada una de estas instancias le asignamos recursos controlados para asegurar su estabilidad: 2048 MB de memoria RAM y 2 vCPUs.

1. Máquina Virtual Gráfica: Pop!_OS 22.04 LTS

Diseñada como la estación de trabajo principal sobre un almacenamiento asignado de 30 GB. Durante la fase de instalación se optó por el particionamiento manual avanzado para segmentar los bloques lógicos de forma eficiente:

Ver Evidencia Fig. 1.1

Ver Evidencia Fig. 1.2

Por medio del comando lsblk se comprobó la correcta creación y montaje de las particiones requeridas sobre el volumen sda:

  • /boot/efi (sda1): 1.0 GiB asignados de manera dedicada al arranque seguro.
  • Raíz / (sda2): 19.5 GiB para los archivos y binarios centrales del S.O. (Formato ext4).
  • swap (sda3): 3.9 GiB como área de paginación o intercambio dinámico (cryptswap).
  • /home (sda4): 5.6 GiB reservados exclusivamente para preservar de manera segura los datos personales del usuario.
Ver Evidencia Fig. 1.3

2. Máquina Virtual de Consola: Rocky Linux 9.4

Instancia minimalista pura (CLI/Línea de comandos) sobre un disco duro virtual de 20 GB. Ofrece un rendimiento optimizado y robusto especializado en servicios empresariales y despliegues de backend.

La distribución de almacenamiento se realizó manualmente bajo el sistema de archivos corporativo xfs de la siguiente manera:

  • Raíz / (sda1): 16.0 GiB para albergar el núcleo del sistema servidor.
  • /home (sda2): 2.0 GiB separados para los directorios de trabajo y scripts.
  • swap (sda3): 2.0 GiB configurados como respaldo de memoria RAM ante sobrecargas.
Ver Evidencia Fig. 1.4

Fase 3: Orquestación con Kubernetes

Ver Entregable

Objetivo de la Fase 3

Implementar y administrar aplicaciones contenerizadas mediante Kubernetes utilizando Minikube como entorno local de pruebas, con el propósito de comprender los conceptos fundamentales de la orquestación de contenedores. Crear y gestionar manifiestos YAML para definir recursos como Deployments y Services, desplegar una aplicación web basada en la imagen oficial de Nginx con múltiples réplicas para garantizar alta disponibilidad, exponer los servicios mediante NodePort para permitir el acceso externo y realizar procesos de escalado manual de los pods. De esta manera, se busca adquirir experiencia práctica en la administración, despliegue, monitoreo y escalabilidad de aplicaciones en infraestructuras modernas basadas en Kubernetes.

Para esta última etapa, elevamos la infraestructura del Grupo 5 hacia un nivel empresarial utilizando Minikube, lo que nos permitió gestionar el ciclo de vida, la alta disponibilidad y el balanceo de carga de nuestros microservicios de forma automatizada.

Conclusiones

    Durante el desarrollo de este proyecto logramos aplicar conocimientos de administración de sistemas Linux, virtualización, Docker y Kubernetes. En la primera fase configuramos la comunicación entre dos máquinas virtuales mediante red y acceso remoto por SSH. En la segunda fase implementamos contenedores Docker para desplegar un frontend con Nginx y un backend con Python. Finalmente, en la tercera fase utilizamos Kubernetes y Minikube para administrar y escalar una aplicación web mediante Deployments y Services. La parte que presentó mayor dificultad fue la fase de Docker, debido a los problemas de configuración de los contenedores, la creación de los Dockerfiles y algunos errores durante la ejecución del backend. También fue necesario realizar varias pruebas hasta lograr que los servicios funcionaran correctamente y pudieran ser accedidos desde el navegador. En conclusión, este proyecto permitió comprender de manera práctica cómo funcionan las tecnologías utilizadas actualmente para la administración, despliegue y escalabilidad de aplicaciones en entornos Linux, fortaleciendo nuestros conocimientos en herramientas fundamentales para el área de sistemas e infraestructura tecnológica.
